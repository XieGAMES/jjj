# Инструкция по запуску настольного приложения

## Подготовка

Перед запуском настольного приложения необходимо:

1. Убедиться, что у вас установлена Java 17 или выше
2. Убедиться, что у вас установлены JavaFX библиотеки (или OpenJDK с JavaFX)
3. Запустить сервер backend приложения

## Запуск сервера backend

1. Откройте терминал или командную строку
2. Перейдите в директорию проекта: `cd C:\JavaProjects\wam`
3. Запустите сервер командой:
   ```
   ./mvnw spring-boot:run
   ```

   Или если вы используете IDE:
   - Запустите класс `zd.zmk.Main` как Spring Boot приложение

## Запуск настольного приложения

### Вариант 1: Использование скрипта (Windows)

1. Убедитесь, что сервер backend запущен
2. Запустите файл `run-desktop.bat` двойным кликом или из командной строки

### Вариант 2: Ручной запуск

1. Убедитесь, что сервер backend запущен
2. Откройте терминал или командную строку
3. Перейдите в директорию проекта: `cd C:\JavaProjects\wam`
4. Скомпилируйте проект: `javac -cp "target/classes;." src/main/java/zd/zmk/desktop/DesktopApplication.java`
5. Запустите приложение: `java -cp "target/classes;." zd.zmk.desktop.DesktopApplication`

## Возможные проблемы и решения

1. **Ошибка "JavaFX is unavailable"**:
   - Установите OpenJDK с JavaFX или добавьте JavaFX библиотеки к пути выполнения
   - Убедитесь, что используете JDK, а не только JRE

2. **Ошибка подключения к API**:
   - Проверьте, что сервер backend запущен на порту 8080
   - Проверьте, что URL в ApiClient.java соответствует адресу сервера

3. **UnsupportedClassVersionError**:
   - Убедитесь, что используете Java 17 для компиляции и запуска
   - Проверьте, что переменная окружения JAVA_HOME указывает на Java 17

4. **ClassNotFoundException**:
   - Убедитесь, что все зависимости скомпилированы и находятся в classpath
