# TASK-001: Проектирование базы данных

## Описание
Спроектировать структуру базы данных для системы управления вендинговыми аппаратами. Привести БД к 3НФ.

## Критерии приемки
- [ ] Выбрана СУБД (PostgreSQL/MySQL/MSSQL)
- [ ] Разработана ER-диаграмма
- [ ] База данных приведена к 3НФ
- [ ] Определены все связи между таблицами

## Основные сущности

### Вендинговые аппараты (vending_machines)
- ID
- Серийный номер (уникальный)
- Инвентарный номер (уникальный)
- Модель
- Тип аппарата (наличные/карта/оба)
- Местоположение
- Фирма-изготовитель
- Страна производства
- Дата изготовления
- Дата ввода в эксплуатацию
- Дата последней поверки
- Межповерочный интервал (месяцы)
- Ресурс ТА (часы)
- Дата следующего ремонта/обслуживания
- Время обслуживания (1-20 часов)
- Статус (Работает/Вышел из строя/В ремонте)
- Дата инвентаризации
- Совокупный доход
- ID сотрудника последней поверки

### Товары (products)
- ID
- Название
- Описание
- Цена
- Количество в наличии
- Минимальный запас
- Склонности к продажам

### Продажи (sales)
- ID
- ID аппарата
- ID товара
- Количество
- Сумма
- Дата и время
- Метод оплаты

### Пользователи (users)
- ID
- ФИО
- Email
- Телефон
- Роль
- Пароль (хеш)

### Обслуживание (maintenance)
- ID
- ID аппарата
- Дата обслуживания
- Описание работы
- Проблемы
- ID исполнителя
