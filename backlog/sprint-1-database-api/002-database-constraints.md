# TASK-002: Реализация ограничений базы данных

## Описание
Реализовать все бизнес-ограничения на уровне SQL для обеспечения целостности данных.

## Критерии приемки
- [ ] Уникальность серийного номера с сообщением об ошибке
- [ ] Уникальность инвентарного номера с сообщением об ошибке
- [ ] Валидация даты ввода в эксплуатацию (между датой изготовления и текущей датой)
- [ ] Валидация даты последней поверки (не раньше даты изготовления, не позже текущей)
- [ ] Валидация даты инвентаризации
- [ ] Ресурс ТА - положительное число
- [ ] Время обслуживания от 1 до 20 часов
- [ ] Статус - ограничение списком значений
- [ ] Межповерочный интервал - положительное число

## Технические требования
- Все ограничения реализуются средствами SQL (CHECK, UNIQUE, TRIGGER)
- Понятные сообщения об ошибках при нарушении ограничений
- Автоматический расчет даты следующей поверки
